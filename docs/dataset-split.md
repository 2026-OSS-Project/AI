# Train / Validation / Test 데이터 분할 기준

## 개요

모델의 성능을 객관적으로 평가하고 과적합(Overfitting)을 방지하기 위해 전체 데이터셋을 학습용(Train), 검증용(Validation), 테스트용(Test) 데이터로 분리한다.

## 데이터 분할 비율

본 프로젝트에서는 다음과 같은 비율로 데이터를 분할한다.

| 데이터셋 | 비율 | 목적 |
| --- | --- | --- |
| Train | 70% | 모델 학습 |
| Validation | 20% | 하이퍼파라미터 조정 및 성능 검증 |
| Test | 10% | 최종 성능 평가 |

- 1차 데이터셋
    | 클래스 | 이미지 개수 | Train | Valid | Test |
    | --- | --- | --- | --- | --- |
    | can | 135 | 94 | 27 | 14 |
    | paper | 125 | 87 | 25 | 13 |
    | plastic | 114 | 79 | 22 | 13 |
    | vinyl | 128 | 89 | 25 | 14 |
    | 총합 | 502 | 349 | 98 | 54 |

클래스별 데이터 수가 불균형해지지 않도록 각 클래스의 비율을 동일하게 유지하는 계층적 분할(Stratified Split)을 적용한다.

## 데이터 분할 원칙

- 동일한 이미지가 여러 데이터셋에 중복 포함되지 않도록 한다.
- Train, Validation, Test 데이터는 상호 독립적으로 구성한다.
- 공공데이터와 직접 수집한 데이터를 통합한 후 분할을 수행한다.
- 클래스별 데이터 비율을 최대한 균등하게 유지한다.

## 데이터셋 구조

```
dataset/
├── train/
│   ├── plastic/
│   ├── vinyl/
│   ├── can/
│   └── paper/
├── valid/
│   ├── plastic/
│   ├── vinyl/
│   ├── can/
│   └── paper/
└── test/
    ├── plastic/
    ├── vinyl/
    ├── can/
    └── paper/
```

## 참고 사항

- 원천데이터(raw data)는 별도로 보관한다.
- 학습에 사용되는 데이터셋은 전처리 및 중복 제거 과정을 거친 후 분할한다.
- 데이터셋 분할 기준은 프로젝트 진행 상황에 따라 변경될 수 있다.
