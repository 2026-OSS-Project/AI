# 데이터셋 폴더 구조

본 프로젝트에서는 데이터 관리의 일관성과 학습 효율성을 높이기 위해 Google Drive 기반으로 데이터셋을 관리하였다.

```
dataset/
├── raw/
│   ├── can/
│   ├── paper/
│   ├── plastic/
│   └── vinyl/
│
├── processed/
│
├── train/
│
├── valid/
│
└── test/
```

## 폴더 설명

- raw
    
    직접 수집한 원본 이미지 저장 폴더
    
- processed
    
    전처리 및 검수 완료 이미지 저장 폴더
    
    - 제거 이미지 조건
        - 흐린 이미지
        - 객체가 잘린 이미지
        - 여러 객체가 포함된 이미지
        - 지나치게 어두운 이미지
        - 저해상도 이미지
    
    이를 통해 학습 데이터의 품질을 일정 수준 이상으로 유지하고자 하였다.
    
- train
    
    모델 학습용 데이터셋
    
- valid
    
    학습 중 성능 검증용 데이터셋
    
- test
    
    최종 성능 평가용 데이터셋 (모델 학습 사용 X / 최종 정확도 측정에만 활용)
